Editing controls

The following controls allow the manipulation of entities in the 3D world while in Edit Mode (F2). The world consists of solid entities and joint entities, which may be organized into static and dynamic bodies. Lolid and joint entities may be combined into dynamic bodies, and the joints of a given body may be linked to other bodies.

Static bodies are highlighted in red, and dynamic bodies are highlighted in green.

The editor features unlimited undo and redo, but these apply only to editing operations (entity creation, deletion, positioning, rotation, embodiment, joining) and not to physical parameter manipulation, entity selection, or transform constraint configuration.

Camera motion scripting controls

The Thumb camera may be programmed to follow a predefined path through a scene, for use in demonstration, movie capture, and debugging.

The script is stored in an XML file in a hidden directory within the user's home directory: ~/.thumb/demo.xml. (To access this directory under Mac OSX, select the Finder menu item “Go/Go To Folder...” and type .thumb.) Thumb does not currently supply a GUI interface to select and save camera script files, and if multiple different scripts are needed then this demo.xml file must by copied and renamed by hand.

The content of the file consists of a series of keyframe camera positions, defined interactively within Thumb using the following keyboard commands.

Shift-PageUpMove the camera to the next keyframe in the script. If the camera is at the last keyframe, it will cycle to the first.
Shift-PageDownMove the camera to the previous keyframe in the script. If the camera is at the first keyframe, it will cycle to the last.
Shift-EndInsert the camera position as a new keyframe, after the current keyframe.
Shift-HomeDelete the current keyframe.
Shift-SpaceBegin playing back the script at the current keyframe. When the script reaches the last keyframe, it will cycle back to the first.

Script playback mode will be automatically enabled after a period of inactivity, by default 2 minutes. During script playback, any interaction will stop playback, leaving the camera in the position where the script was stopped.

Calibration controls

This section describes the calibration control bindings hard-coded in Thumb. The number of complexity of these may at first seem ridiculous. However, these bindings are not active under normal circumstances, and are hidden behind an explicit calibration mode. They are here for experts, and never need to be touched by the vast majority of users.

Calibrating a Varrier or a multi-projector dome can be an arduous task. Years of experience have shown that a solid set of keyboard bindings make the performance of these tasks far quicker and easier than any GUI interface. While it takes longer for the beginner calibrator to learn to perform a calibration, the keyboard allows the expert calibrator to work very quickly without the need to shift attention from the display to the interface.

Calibration state controls

Control-TabToggle calibration mode. In calibration mode, the normal distribution of events is interrupted. Instead of being sent to the application, events are routed instead to the application configuration. This allows view frusta and display modes to be tuned interactively.
Control-SpaceCycle forward through the list of configurable displays.
Control-BackspaceCycle backward through the list of configurable displays.
